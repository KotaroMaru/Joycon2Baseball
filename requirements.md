## 1. プロジェクト概要

### コンセプト
大谷翔平投手の160km/h級の球を打者視点で体験する没入型野球ゲーム。  
Joy-Con 2を実際に振ってバットスイングを行い、ホームランの快感を体感する。

### ターゲット体験
- 160km/hのストレートを実際に体で感じるリアルな打席体験
- ホームランを打った瞬間の爽快感・達成感
- 本格野球ルール（カウント・ランナー・得点）によるゲームの深み

### キーワード
`没入感` `体感型` `ガチ寄り` `本格野球ルール` `シンプルなゲームフロー`

## 4. ゲームシステム詳細

### 4.1 カウント・ルール管理

#### ストライクカウント
- スイングして空振り → ストライク
- 見逃し（ストライクゾーン通過・スイングなし） → ストライク
- 3ストライクで三振 → 1アウト

#### ボールカウント
- ストライクゾーン外を見逃し → ボール
- 4ボールで四球 → 出塁（ランナー1塁へ）

#### ファールの扱い
- 本物ルール通り
- 2ストライク未満のファール → ストライクカウント増加
- **2ストライク後のファール → ストライクにならない（カウント維持）**

#### アウトカウント
- 三振 → 1アウト
- 打球がアウトエリアに落下 → 1アウト
- 3アウトでゲーム終了 → スタッツ画面へ

### 4.2 球速と体感時間

| 球速 | マウンドからホームベースまでの時間（約18m） |
|:---:|:---:|
| 80 km/h | 約0.81秒 |
| 120 km/h | 約0.54秒 |
| 145 km/h | 約0.45秒 |
| 160 km/h | 約0.405秒 |

**実装方針**: 球速スライダーの値からRigidbodyの初速を逆算して設定。

### 4.3 ボールの視認性

- **本物と同じ体感を目標とする**
- リリース直後（投手の手を離れた瞬間）からボールが見え始める
- 160km/hでは約0.4秒で到達するため、見てから振るのが非常に困難
- 球速スライダーで80km/hに下げることで初心者でも体験可能

## 5. Joy-Con入力・スイング検出

### 5.1 使用するAPI（Joycon2Manager）

```csharp
// スイング開始検出（加速度のピーク値）
Vector3 accel = Joycon2Manager.Instance.RightAccel;

// バットの軌道追従（フレーム間の回転積分）
Vector3 gyroDelta = Joycon2Manager.Instance.ConsumeGyroDelta(1); // 1 = Right

// 重力補正済み加速度
Vector3 linearAccel = Joycon2Manager.Instance.GetLinearAcceleration(1, currentRotation);
```

### 5.2 スイング検出アルゴリズム

**方針**: 加速度でスイング開始を検出し、その後はジャイロで軌道を追う（ハイブリッド方式）

```
1. RightAccelの大きさが閾値（TBD）を超えたらスイング開始と判定
2. スイング中はConsumeGyroDelta()でフレームごとの回転量を積分
3. バットのTransform.rotationをリアルタイム更新
4. スイング終了（加速度が閾値以下に戻る）で検出終了
```

**閾値はゲーム内バランスを見ながら調整（実装時に策定）**

### 5.3 バットの表示

- 打者視点の手元にバット3Dモデルを配置
- Joy-Conの動きにリアルタイムで追従して表示
- `ConsumeGyroDelta()`を使って1:1の回転同期を実現（仕様書記載の重要APIを使用）
- バットCollider: CapsuleCollider（シンプルな1本のCollider）

### 5.4 キャリブレーション

#### 通常キャリブレーション（ゲーム開始時）
- キャリブレーション画面で「Joy-Conを水平に構えてください」と指示
- AボタンまたはBボタンで確定
- その瞬間のジャイロ・加速度値をゼロ点として記録
- 球速スライダーもこの画面で設定

#### 打席中の再キャリブレーション
- 投球間隔中（ボールが飛んでいない時間）に任意のボタン長押しで呼び出し可能
- 「水平に構えてください」のガイドテキストを一瞬表示後、ボタンで確定
- v1はデバッグ用途も兼ねる

## 6. 投球システム

### 6.1 球種一覧

| 球種 | 特徴 | v1実装 |
|:---|:---|:---:|
| ストレート | 160km/h。完全直線 | ✅ |
| スプリット | 大谷の決め球。沈む変化球 | ✅ |
| スライダー | 横に滑る変化球 | ✅ |
| カーブ | 大きく縦に曲がる | ✅ |
| カットボール | 小さく鋭く変化 | ✅ |

### 6.2 球の軌道実装方式（ハイブリッド方式）

```
ストレート:
  → Rigidbodyに初速Vector3を与えて物理演算に任せる
  → 重力はUnity標準(9.81m/s²)を使用し、ゲームバランスを見て調整

変化球:
  → Rigidbodyで初速を与えた後、途中からAddForce()で変化方向に力を加える
  → スプリット: 下方向にAddForce
  → スライダー: 横方向にAddForce
  → カーブ: 縦＋横にAddForce（大きく曲げる）
  → カットボール: 小さく横方向にAddForce
```

**変化量・物理パラメータはゲーム内バランスを見ながら調整**

### 6.3 投球のトリガー

- 打者がAボタン（またはBボタン）を押したら投球開始
- 自分のペースで打てる
- v1配球AI: ランダムに球種・コースを選択

### 6.4 ストライクゾーン判定

- 9分割ゾーン（高め/真ん中/低め × 内/真ん中/外）を設定
- ボールがゾーン内を通過 → ストライク
- ゾーン外を通過 → ボール
- シンプルな矩形判定（BoxCollider）

### 6.5 見逃し後のボール処理

- スイングしなかった・当たらなかった場合
- ボールはホームベースを通過しキャッチャーミットまで飛ぶ
- ミットに収まる演出後、次の投球待機状態へ

## 7. 打球判定システム

### 7.1 衝突判定

```
バットCollider（CapsuleCollider） × ボールCollider（SphereCollider）
  → OnCollisionEnter()で衝突を検出
  → 衝突時のRelativeVelocityとバットの回転情報からRigidbodyが打球ベクトルを自動計算
  → 追加パラメータ: スイング開始タイミング（ボール到達との差分）でパワー補正
```

### 7.2 打球パラメータへの変換

| Joy-Conの動き | 打球への影響 |
|:---|:---|
| 振る速度（加速度ピーク値） | 打球の初速（飛距離） |
| ジャイロの上下角度 | 打球の上下角度（フライ/ゴロ） |
| ジャイロの左右角度 | 打球の左右方向 |
| タイミング（早い/遅い） | インコース/アウトコース方向の補正 |
| ミート位置 | v1では影響なし（シンプル実装） |

### 7.3 フィールドエリア設定

野球盤風にフィールドを3Dで設定し、打球の落下点でエリア判定。

```
フィールドエリア（Collider/Triggerで設定）:
  ├── HRエリア: フェンス越え（外野フェンス外）
  ├── 2Bエリア: 外野深め（外野フェンス手前の一定範囲）
  ├── 1Bエリア: 内野外縁〜外野浅め
  ├── OUTエリア: 内野（守備がいる想定、アウトになる）
  └── FAULエリア: ファールライン外
```

**判定方式**: 打球GameObjectのTriggerEnter()でエリアを判別

### 7.4 打球結果テキスト表示

| 結果 | 表示テキスト |
|:---:|:---:|
| ホームラン | `HOMERUN!` |
| 2塁打 | `2 BASE HIT!` |
| 1塁打 | `HIT!` |
| アウト | `OUT!` |
| 空振り | `STRIKE!` |
| ファール | `FOUL!` |
| 四球 | `BALL FOUR!` |

## 8. ランナー・得点管理

### 8.1 ランナー管理ルール

#### 出塁条件
- 1塁打（1B）: ランナーは1塁へ
- 2塁打（2B）: ランナーは2塁へ
- ホームラン（HR）: 全ランナー生還（得点加算）
- 四球（BB）: ランナーは1塁へ

#### ヒット時のランナー進塁
- **1Bヒット**: 全ランナーは2塁分進塁
- **2Bヒット**: 全ランナーは3塁分進塁（3塁ランナーは生還）
- **HR**: 打者含む全ランナーが生還

#### ホームイン（得点）
- ランナーが本塁に到達したら得点+1
- スコアボードの得点欄をリアルタイム更新

### 8.2 得点表示

- ホームインの瞬間にポップアップで`+1`を表示
- スコアボードの得点欄に累積表示

## 9. UI・HUDデザイン

### 9.1 打席中のスコアボード（画面左上）

野球中継風のスコアボードを画面左上に常時表示。

```
┌─────────────────────────┐
│  B  S  O     得点        │
│  2  1  1      3          │
│  ┌─┐ ┌─┐ ┌─┐            │
│  │  │ │■│ │  │   ← 塁ランプ │
│  └─┘ └─┘ └─┘            │
└─────────────────────────┘
```

**表示内容**:
- **B**: ボールカウント（0〜3）
- **S**: ストライクカウント（0〜2）
- **O**: アウトカウント（0〜2）
- **得点**: 累積得点
- **塁ランプ**: 1塁・2塁・3塁のランナー有無（■=あり、□=なし）

**デザイン方針**:
- フォント: 日本語対応の太めのゴシック体（読みやすさ重視）
- 背景: 半透明の黒パネル（没入感を損なわないよう控えめに）

### 9.2 ストライクゾーンUI

- 画面端（カウントと一緒に）に常時小さく表示
- 9分割グリッドで現在のゾーンを可視化
- 投球されたボールの通過位置を一瞬ハイライト

### 9.3 打席中のデバッグUI（v1限定）

- 球速スライダー（打席中にいつでも変更可能）
- 開発・デバッグ用。v2で削除またはオプション化

## 10. 演出・サウンド

### 10.1 サウンド方針

- 球場の環境音＋観客の歓声のみ（BGMなし・没入重視）
- 素材調達: フリー素材を使用

### 10.2 サウンドイベント一覧

| イベント | サウンド |
|:---|:---|
| 投球中 | 球の風切り音（ヒュッ） |
| ヒット | バット打撃音（カキーン）＋観客歓声 |
| ホームラン | 打撃音＋大歓声（特別ver） |
| 空振り | バット空切り音（ビュッ） |
| ファール | 軽い打撃音＋小さな歓声 |
| ストライク（見逃し） | キャッチャーミットの捕球音 |
| ボール | キャッチャーミットの捕球音（小さめ） |
| 三振 | 審判の「ストライク！」SE |
| アウト | 審判の「アウト！」SE |
| 四球 | 小さな歓声 |

### 10.3 HD振動（v2以降に延期）

v1では振動は一切なし。v2で以下を実装予定：

| イベント | 振動パターン |
|:---|:---|
| ヒット | 短く激しいバースト振動（インパクト感重視） |
| ホームラン | ヒット時より強い打撃振動 → 余韻の低周波振動（1秒程度） |
| 空振り（ストライク） | 短いショートバイブレーション |
| ファール | クッと短いパルス |
| 見逃しストライク | 小さくプシュッとした短い振動 |
| ボール判定 | 振動なし |
| 投球中の風切り | 微細な高周波振動（ボールが近づくにつれ強くなる） |

### 10.4 打撃結果演出

#### ホームラン演出シーケンス
1. バット・ボール衝突の瞬間: 通常の打撃音＋大歓声
2. 打球が高く飛んだ瞬間: スロー演出（Time.timeScale下げ）
3. `HOMERUN!` テキストを画面中央に大きく表示
4. フェンス越えを確認後: リプレイカメラへ自動切替
5. **リプレイカメラ**: ホームベース内野側のアングルから打球が消えるまで追う
6. リプレイ終了後: スタッツ画面へ

#### その他の結果演出
- ヒット・アウト: 結果テキストを画面に大きく表示 → 即座に次の投球へ
- 三振: `STRIKE OUT!` テキスト → アウトカウント更新 → 次の打席へ

## 11. ゲームフロー

### 11.1 全体フロー

```
タイトル画面
  └─→ [球速スライダー設定] + [スタートボタン]
        └─→ キャリブレーション画面
              │ 「Joy-Conを水平に構えてください」
              │ [Aボタンで確定]
              └─→ 打席開始
                    │
                    ├─→ [Bボタンで投球開始]
                    │     ├─→ スイング → 打球判定 → 結果演出
                    │     ├─→ 見逃し → ストライク/ボール判定
                    │     └─→ 次の投球へ（繰り返し）
                    │
                    ├─→ [3アウト] → スタッツ画面
                    │
                    └─→ 打席中キャリブレーション（任意）
                          └─→ 特定ボタン長押し → 再キャリブ → 打席再開

スタッツ画面
  ├─→ [リトライ] → タイトルに戻る
  ├─→ [タイトル] → タイトルに戻る
  └─→ [球速設定] → 球速スライダーを表示して変更
        └─→ [リトライ] → タイトルに戻る
```

### 11.2 画面一覧

| 画面名 | 内容 | 主なUI要素 |
|:---|:---|:---|
| タイトル画面 | ゲーム開始 | タイトルロゴ・球速スライダー・スタートボタン |
| キャリブレーション画面 | Joy-Conゼロ点設定 | 説明テキスト・球速スライダー・確定ボタン |
| 打席画面 | メインゲームプレイ | スコアボード（左上）・ストライクゾーン・デバッグ球速スライダー |
| ホームランリプレイ | 打球追いかけカメラ | なし（没入感優先） |
| スタッツ画面 | 打席結果一覧 | 結果リスト・リトライ/タイトル/球速設定ボタン |

### 11.3 画面遷移

- v1: **カット（即時切り替え）**
- v2以降: フェードイン/アウト・スライドアニメーション等を追加予定

### 11.4 スタッツ画面の表示内容

- 各打席の結果リスト（例: `1打席目: HR`, `2打席目: 1B`, `3打席目: 三振`）
- 打席数・打率・長打率・本塁打数・最長飛距離

## 13. 技術仕様・実装メモ

### 13.1 Joycon2-Unity 連携

```csharp
// 接続確認
bool connected = Joycon2Manager.Instance.rightConnected;

// スイング検出（Update内）
void Update() {
    Vector3 accel = Joycon2Manager.Instance.RightAccel;
    Vector3 gyroDelta = Joycon2Manager.Instance.ConsumeGyroDelta(1);
    
    // バット回転を積分
    batRotation *= Quaternion.Euler(gyroDelta);
    bat.transform.rotation = calibrationOffset * batRotation;
    
    // スイング開始検出
    if (accel.magnitude > swingThreshold && !isSwinging) {
        isSwinging = true;
        OnSwingStart();
    }
}
```

> ⚠️ **注意**: `LeftGyro`/`RightGyro`は最新1パケットのみ。回転同期は必ず`ConsumeGyroDelta()`を使用すること（仕様書セクション5参照）

void OnCalibrationConfirmed() {
    // 現在のJoy-Con姿勢をゼロ点として記録
    calibrationOffset = Quaternion.Inverse(currentJoyConRotation);
